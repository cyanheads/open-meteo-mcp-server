# 0.2.3 — 2026-07-11

## Changed

- **`openmeteo_dataframe_query` caps inline `rows` to a 100-row preview** (previously up to the canvas's row limit, default 10,000). `content[]` and `structuredContent.rows` now carry an identical preview; `row_count` still reports the true total, and the response points at SQL `LIMIT`/`OFFSET` to page beyond the preview. ([#16](https://github.com/cyanheads/open-meteo-mcp-server/issues/16))
- **Dependencies:**
  - `@biomejs/biome` `^2.5.2` → `^2.5.3`
  - `@types/node` `^26.1.0` → `^26.1.1`
  - `tsc-alias` `^1.8.17` → `^1.9.0`
  - `vitest` `^4.1.9` → `^4.1.10`
  - `typescript` held at `^6.0.3` (latest 7.0.2 is a deliberate major hold) — added to `devcheck.config.json`'s `outdated.allowlist`

## Fixed

- **`content[]` renders every row `structuredContent` returns, across all seven time-series tools.** `openmeteo_get_forecast`, `openmeteo_get_air_quality`, `openmeteo_get_marine`, and `openmeteo_get_flood` dropped their preview caps (48/30 rows) and "…and N more" lines; `openmeteo_get_historical`, `openmeteo_get_ensemble`, and `openmeteo_get_climate` now render the full spillover preview array on the truncated path instead of re-slicing it to a smaller display cap. Text-only MCP clients no longer silently lose rows that `structuredContent` already carried. ([#16](https://github.com/cyanheads/open-meteo-mcp-server/issues/16))
- **`openmeteo_get_flood` and `openmeteo_get_climate` accept empty or omitted `daily_variables`.** Dropped `.min(1)` in favor of `.optional()` on the input schema, so calls reach the existing handler guard and its declared `no_variables_requested` recovery instead of a generic Zod `-32602` validation error. ([#17](https://github.com/cyanheads/open-meteo-mcp-server/issues/17))
- **`openmeteo_get_historical`, `openmeteo_get_ensemble`, and `openmeteo_get_climate` return the exact staged `table_name`** alongside `canvas_id` when a result spills to DataCanvas — previously the spillover helper's randomly generated table name was discarded, so `openmeteo_dataframe_query`'s documented `spilled_<id>` naming convention never matched the real table and every spill→query round trip failed with `missing_table`. `openmeteo_dataframe_query`'s description and `missing_table` message now point at `table_name` (with `openmeteo_dataframe_describe` as a fallback) instead of implying the name derives from `canvas_id`. ([#18](https://github.com/cyanheads/open-meteo-mcp-server/issues/18))
