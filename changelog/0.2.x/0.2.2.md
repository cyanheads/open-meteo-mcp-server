# 0.2.2 — 2026-07-06

## Added

- **`openmeteo_geocode` gains a `country` input.** ISO 3166-1 alpha-2 code, regex-validated and uppercased before the upstream call, mapped to the `countryCode` filter — narrows ranked matches to one country when a name is shared (e.g. Paris, FR vs. Paris, TX). ([#15](https://github.com/cyanheads/open-meteo-mcp-server/issues/15))

## Fixed

- **`openmeteo_geocode` description and `no_results` recovery no longer imply `country`/`admin1` are inputs.** Both are output-only result fields for disambiguating among matches. Callers are steered to bare place names instead of folding in a qualifier (`"Baoding"`, not `"Baoding Hebei"`); the `no_results` recovery now distinguishes an administrative qualifier (drop it and retry) from a physical feature or landmark outside the populated-places gazetteer (query the nearest town instead). ([#15](https://github.com/cyanheads/open-meteo-mcp-server/issues/15))
- **README corrected** — the `openmeteo_geocode` section claimed no-match returns an empty `results` array; it throws `no_results`.
