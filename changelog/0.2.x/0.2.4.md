# 0.2.4 — 2026-07-15

## Fixed

- **Spill eligibility is serialized payload size, not row count.** `openmeteo_get_historical`, `openmeteo_get_ensemble`, and `openmeteo_get_climate` dropped their `INLINE_LIMIT = 500` row gate for `exceedsInlineBudget()`, which measures the records against the same `PREVIEW_CHARS` budget handed to `spillover()` as `previewChars` — so the precheck and the helper never disagree. Wide results under 500 rows (a 16-day ensemble fan-out is ~376 KB in 384 rows) previously returned inline with no `canvas_id` and no retrieval path. ([#23](https://github.com/cyanheads/open-meteo-mcp-server/issues/23))
- **Spilled numeric columns keep their type through a leading run of null rows.** The three tools now pass `spillover()` an explicit `schema` from `deriveSpillSchema()`, derived from every staged row rather than the helper's own preview buffer. An `openmeteo_get_ensemble` `past_days` response opens with all-null placeholder rows the models don't hindcast, which exhausted the sniff window and left every member column `VARCHAR` — `AVG()` then failed with `No function matches avg(VARCHAR)` unless the caller cast each value. ([#21](https://github.com/cyanheads/open-meteo-mcp-server/issues/21))
- **Daily-only columns survive a mixed hourly + daily spill.** `openmeteo_get_historical` and `openmeteo_get_ensemble` concatenate hourly records ahead of daily ones, so a large hourly pull exhausted the sniff window before a daily row was sampled and the daily column was never created on the table. The same full-record schema unions both cadences into the one staged table, keeping the existing single `table_name` output; separate by timestamp shape in SQL (`WHERE time LIKE '%T%'` — hourly is `YYYY-MM-DDTHH:MM`, daily is `YYYY-MM-DD`). ([#22](https://github.com/cyanheads/open-meteo-mcp-server/issues/22))

## Changed

- **`canvas_id` and `truncated` descriptions no longer claim a ~500-record threshold** across the three spill-capable tools, `README.md`, and `docs/design.md`. They now state that a result spills when it is too large to return inline.

## Dependencies

- `ignore` ^7.0.5 → ^7.0.6
