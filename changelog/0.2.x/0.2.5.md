# 0.2.5 — 2026-07-16

## Added

- **`openmeteo_get_flood` DataCanvas spillover.** A wide GloFAS reanalysis pull (a full 1984–present history is ~15.5k daily records) now spills to a DuckDB canvas instead of returning inline, matching `openmeteo_get_historical`, `openmeteo_get_ensemble`, and `openmeteo_get_climate`. Adds an optional `canvas_id` input and `record_count`, `truncated`, `canvas_id`, and `table_name` output fields — `daily` stays required, so every change is additive. Spill eligibility is serialized payload size (never a row count), and the staged schema is derived from every record so a leading run of GloFAS nulls cannot type a numeric column `VARCHAR`. Query the staged table with `openmeteo_dataframe_query`. ([#19](https://github.com/cyanheads/open-meteo-mcp-server/issues/19))

## Fixed

- **`openmeteo_get_flood` rejects conflicting date/mode inputs before the API call.** A one-sided `start_date`/`end_date` now fails `date_range_incomplete`, and `forecast_days` combined with a date range fails `forecast_days_conflict` — reported first when both faults are present, since only it names the caller's chosen mode. Each carries a targeted recovery, and the tool description and README no longer claim `forecast_days` and a date range can be combined. ([#25](https://github.com/cyanheads/open-meteo-mcp-server/issues/25))
- **`openmeteo_dataframe_query` renders struct and list cells faithfully.** Object and array cells are `JSON.stringify`-serialized — structs as `{"k":v}`, lists with their brackets intact — instead of `[object Object]` and a bare comma join; primitives stay on the `String()` path. Every cell escapes a literal `|` so a value cannot break the markdown table. `content[]` now carries the same values as `structuredContent.rows`. ([#20](https://github.com/cyanheads/open-meteo-mcp-server/issues/20))
- **`formatUnits` keeps the `time` entry in `content[]`.** The rendered units line no longer drops `time: iso8601`, so text-only clients read the same units map as `structuredContent.hourly_units` / `daily_units`. ([#24](https://github.com/cyanheads/open-meteo-mcp-server/issues/24))
