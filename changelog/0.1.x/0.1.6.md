# 0.1.6 — 2026-06-20

Framework-maintenance release: adopts `@cyanheads/mcp-ts-core` `^0.10.9` and re-syncs the vendored scripts and skills. No tool, schema, or behavior changes.

## Changed

- **`devcheck` — `Dependency Specifiers` gate.** New `scripts/check-dependency-specifiers.ts` (flag `--no-dep-specifiers`) hard-fails on floating specifiers in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map: `latest` is rejected everywhere, and `*` / pre-release dist-tags (`next`/`beta`/`canary`/`rc`) are rejected in `dependencies`/`devDependencies` but allowed in `peerDependencies`/`optionalDependencies`. Catches `bun update --latest` writing a literal `latest` dist-tag into the lock's workspace map. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **`devcheck` — plugin marketplace manifest checks.** `scripts/lint-packaging.ts` gains check 10, validating `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` for non-empty descriptions, unscoped machine name in display fields, and the full scoped `package.json` name in the `npx -y` install arg. Gated by the new `devcheck.config.json` `packaging.pluginManifests` flag (set `true`); the `Packaging` gate now also runs when a plugin manifest is present. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`.codex-plugin/plugin.json`** — `interface.longDescription` populated (was empty) to satisfy the new non-empty-description manifest check.
- **`devcheck` fresh-scaffold guards** synced — `build-changelog.ts --check`, `devcheck.ts`, and `check-framework-antipatterns.ts` skip git-dependent and changelog-sync checks when run outside a git repo or before any version files exist; `check-skill-versions.ts` skips a `SKILL.md` deleted from the worktree instead of crashing on ENOENT. ([cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243), [#237](https://github.com/cyanheads/mcp-ts-core/issues/237))
- **Vendored skills re-synced to 0.10.9** — `api-context` documents `ctx.content`, the always-present collector for image/audio blocks that prepend to `content[]` and never enter `structuredContent` ([cyanheads/mcp-ts-core#239](https://github.com/cyanheads/mcp-ts-core/issues/239)); `api-canvas` documents the SQL gate classifying a parse-OK / prepare-fail `SELECT` as `invalid_sql` with the DuckDB binder detail ([cyanheads/mcp-ts-core#236](https://github.com/cyanheads/mcp-ts-core/issues/236)); `add-tool`, `api-config`, `polish-docs-meta`, `git-wrapup`, `orchestrations`, and the seven body-drift skill bumps (`api-auth`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `report-issue-local`, `tool-defs-analysis`) follow. ([cyanheads/mcp-ts-core#238](https://github.com/cyanheads/mcp-ts-core/issues/238))
