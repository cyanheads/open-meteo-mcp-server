# 0.1.3 — 2026-06-02

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.16` → `^0.9.21` — picks up three framework fixes:
  - HTTP transport per-request log context: each request now carries fresh request + trace/span IDs instead of the frozen boot context.
  - **`fetchWithTimeout`** strips query-string secrets (e.g. `?api_key=`) from error messages and logs.
  - **`withRetry`** fails fast on non-retryable errors; `ctx.fail` auto-populates the `retryable` flag.
- **Client config key** renamed from `"open-meteo"` to `"open-meteo-mcp-server"` in README install snippets (hosted, bunx, npx, Docker).
- **`scripts/devcheck.ts`** updated to current framework template.
- **`scripts/release-github.ts`** and **`scripts/check-skill-versions.ts`** added (framework script sync).
- **Skills** (`add-tool`, `add-service`, `api-canvas`, `api-context`, `api-linter`, `api-utils`, `design-mcp-server`, `release-and-publish`) updated to framework 0.9.21; `api-mirror` and `orchestrations` skills added.
- **`CLAUDE.md` / `AGENTS.md`** skill table and commands section updated to reflect new skills and `release:github` script.
