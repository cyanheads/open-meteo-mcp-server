# 0.1.4 — 2026-06-04

## Added

- **`openmeteo_get_ensemble`** — probabilistic ensemble weather forecast returning per-member hourly/daily time series (up to 51 members, 16 days ahead). Supported models: `ecmwf_ifs025`, `gfs025`, `icon_seamless`, `gem_global`. Large multi-member pulls spill to DataCanvas when `CANVAS_PROVIDER_TYPE=duckdb`. Resolves [#4](https://github.com/cyanheads/open-meteo-mcp-server/issues/4).
- **`openmeteo_get_flood`** — GloFAS river discharge forecast (up to 210 days) and reanalysis (1984–present) via the Open-Meteo Flood API. Coordinate-based; no river ID required. Resolves [#3](https://github.com/cyanheads/open-meteo-mcp-server/issues/3).
- **`EnsembleParams` / `EnsembleEnvelope` / `FloodParams` / `FloodEnvelope`** — new types in `open-meteo-service.ts` and `types.ts` supporting the two new endpoints.
- **`OPEN_METEO_ENSEMBLE_BASE_URL` / `OPEN_METEO_FLOOD_BASE_URL`** — optional env vars to override the ensemble and flood API base URLs (for testing or self-hosted deployments).
