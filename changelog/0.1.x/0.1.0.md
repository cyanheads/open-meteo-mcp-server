# 0.1.0 — 2026-05-30

Initial release of open-meteo-mcp-server. All data from Open-Meteo.com — keyless, no API key required for non-commercial use (CC BY 4.0).

## Added

- **`openmeteo_geocode`** — resolves place names to ranked coordinate matches with country, region, elevation, timezone, and population; prerequisite for name-based weather queries.
- **`openmeteo_get_forecast`** — hourly/daily weather forecast up to 16 days ahead; also covers up to 92 days of recent past via `past_days`; all standard weather model variables (temperature, precipitation, wind, humidity, cloud cover, UV index, etc.).
- **`openmeteo_get_historical`** — ERA5 reanalysis archive from 1940 to present; same variable names as forecast for direct comparison; large date ranges (multi-year hourly) spill to DataCanvas for SQL querying when `CANVAS_PROVIDER_TYPE=duckdb`.
- **`openmeteo_get_marine`** — marine wave/swell forecast up to 7 days for coastal and ocean coordinates; hourly and daily variables (wave height, period, direction, wind-wave, swell, SST).
- **`openmeteo_get_air_quality`** — CAMS modeled air quality forecast (no historical archive); PM2.5, PM10, ozone, NO₂, SO₂, CO, dust, pollen, European/US AQI indices.
- **`openmeteo_get_elevation`** — Copernicus DEM terrain elevation (~90m resolution) for up to 100 coordinate pairs per call.
- **Handler-side columnar→per-timestamp reshape** (`reshape-utils.ts`) — converts the Open-Meteo columnar response format into per-timestamp records across all weather and marine tools.
- **Geocode-before-forecast workflow** — server-level instructions and tool descriptions guide clients through the `openmeteo_geocode` → weather tool two-step.
- **DataCanvas spillover** for `openmeteo_get_historical` — inline result cap of 500 records; larger queries register a DuckDB table for downstream SQL (`SELECT … FROM historical_data`).
- **Typed error contracts** on all tools — `no_results`, `date_out_of_range`, `date_order_invalid`, `no_variables_requested`, `invalid_variable`, `coordinate_count_mismatch`; each with recovery hints.
- **50 tests** across 7 test files — unit coverage for handlers, reshape logic, error contracts, sparse payloads, and DataCanvas spillover path.
