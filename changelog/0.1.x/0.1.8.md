# 0.1.8 — 2026-07-02

## Changed

- **Unknown-variable errors lead with the fix, not upstream jargon.** The six weather/environment tools (`openmeteo_get_forecast`, `openmeteo_get_historical`, `openmeteo_get_marine`, `openmeteo_get_air_quality`, `openmeteo_get_ensemble`, `openmeteo_get_flood`) frame the upstream Swift type-init rejection (`Data corrupted at path ''…`) via a shared `frameInvalidVariableMessage` helper — the message names the offending value(s) when the upstream embeds them and demotes the raw string to a trailing parenthetical. When the upstream echoes a whole comma-joined list (valid names included), the message says at least one is invalid rather than claiming all are. ([#7](https://github.com/cyanheads/open-meteo-mcp-server/issues/7))
- **Dataframe tools document both producers.** `openmeteo_dataframe_describe` / `openmeteo_dataframe_query` descriptions, `canvas_id` input docs, and `canvas_not_found` recovery hints now name `openmeteo_get_ensemble` alongside `openmeteo_get_historical` as canvas sources. ([#8](https://github.com/cyanheads/open-meteo-mcp-server/issues/8))

## Fixed

- **Declared recovery hints reach the wire.** Every `ctx.fail` site now passes `ctx.recoveryFor(reason)`, so the contract's recovery guidance ships as `data.recovery.hint` (and into `content[]` text) instead of staying declaration-only. ([#11](https://github.com/cyanheads/open-meteo-mcp-server/issues/11))
- **`canvas_id` returned only when `truncated: true`.** `openmeteo_get_historical` / `openmeteo_get_ensemble` no longer surface a `canvas_id` pointing at an empty canvas when the spillover path runs but stays under its byte threshold (`spilled: false`). ([#8](https://github.com/cyanheads/open-meteo-mcp-server/issues/8))
- **Unknown-canvas errors carry the right guidance.** The dataframe tools rethrow the framework `acquire()` NotFound under their own `canvas_not_found` contract — the framework's generic "omit canvas_id to start a new canvas" hint was wrong for tools where `canvas_id` is required and never created. ([#8](https://github.com/cyanheads/open-meteo-mcp-server/issues/8))
