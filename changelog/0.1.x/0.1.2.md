# 0.1.2 — 2026-05-31

## Added

- **`openmeteo_dataframe_query`** — run read-only SELECT against tables staged on a DataCanvas by `openmeteo_get_historical` when `truncated: true` ([#1](https://github.com/cyanheads/open-meteo-mcp-server/issues/1)).
- **`openmeteo_dataframe_describe`** — list tables and column schemas on a staged canvas; call before querying ([#1](https://github.com/cyanheads/open-meteo-mcp-server/issues/1)).
- **`@duckdb/node-api` ^1.5.3-r.3** — native DuckDB binding required for `CANVAS_PROVIDER_TYPE=duckdb` to initialize correctly.

## Fixed

- **Dead canvas** — `openmeteo_get_historical` staged data to a canvas but exposed no tool to query it; the two new tools close that gap ([#1](https://github.com/cyanheads/open-meteo-mcp-server/issues/1)).
- **README `secedgar_dataframe_query` reference** — the `openmeteo_get_historical` section incorrectly pointed users at a different server's tool; updated to `openmeteo_dataframe_query`.
- **`CANVAS_PROVIDER_TYPE` default mis-documented** — README configuration table listed the default as `duckdb`; corrected to `none` (framework default; spillover disabled until explicitly set).
