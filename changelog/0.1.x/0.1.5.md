# 0.1.5 — 2026-06-12

## Added

- **`Dockerfile` healthcheck** — `HEALTHCHECK` polls `/healthz` via a bun-native `fetch` (the slim runtime ships no curl/wget). The `org.opencontainers.image.version` label is now stamped from an `APP_VERSION` build arg.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into the `bundle` script: runs `mcpb clean`, strips dependency-shipped agent-doc trees (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) nested under `node_modules/` that root-anchored `.mcpbignore` patterns can't reach, then asserts none remain.

## Changed

- **`createApp()`** now sets explicit `name` and `title` of `open-meteo-mcp-server`; `.codex-plugin/plugin.json` `interface.displayName` corrected to the bare repo name so the display identity matches across surfaces.
- **`.mcpbignore`** patterns root-anchored with a leading `/` so they match only top-level paths.
- **Vendored skills** re-synced to the 0.10.6 framework (`ctx.sample` removed from the Context reference; `check-framework-antipatterns.ts` gains a `z.coerce.boolean()` env-flag rule).
- **Dependencies:**
  - `@cyanheads/mcp-ts-core` `^0.9.21` → `^0.10.6`
  - `@biomejs/biome` `^2.4.16` → `^2.5.0`
  - `@types/node` `^25.9.1` → `^25.9.3`
  - `typescript` `^5.9.3` → `^6.0.3`
  - `tsc-alias` `^1.8.16` → `^1.8.17`
  - `vitest` `^4.1.0` → `^4.1.8`

## Security

- **`openmeteo_dataframe_query`** now executes with `denySystemCatalogs: true`. SQL referencing a system catalog (`information_schema`, `sqlite_master`, `pg_catalog`, or a `duckdb_*()` function) is rejected with the new `system_catalog_access` error so a caller cannot enumerate other staged canvases; `openmeteo_dataframe_describe` remains the supported way to list tables and columns.
